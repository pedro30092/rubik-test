# Implementation Plan: Phase 1 — Backend Exam Session API

**Branch**: `001-backend-exam-api` | **Date**: 2026-04-02 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-backend-exam-api/spec.md`

## Summary

Build two serverless API endpoints — `POST /v1/exams` (create exam session) and `PUT /v1/exams/:sessionId` (finish exam session) — backed by AWS Lambda (TypeScript/Node.js 20) and a single DynamoDB table (`statistics`). Deployed manually to API Gateway HTTP API (v2) in a dev environment. No authentication. A hardcoded default user is used throughout.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js 20.x  
**Primary Dependencies**: `@aws-sdk/lib-dynamodb` (DocumentClient), `esbuild` (build), `@types/aws-lambda`  
**Storage**: AWS DynamoDB — single table `statistics` (PK: `EXAM_STATS`, SK: `<ISO-UTC>#<4-char-hex>`)  
**Testing**: Manual — curl collection + AWS CLI DynamoDB inspection (see `quickstart.md`)  
**Target Platform**: AWS Lambda Node.js 20.x runtime, API Gateway HTTP API v2  
**Project Type**: Serverless web-service  
**Performance Goals**: Cold start < 1 second (esbuild bundle, no webpack)  
**Constraints**: HTTP API v2 only (no REST API); manual infrastructure setup (no IaC); dev environment only  
**Scale/Scope**: Single user, 2 endpoints, 1 DynamoDB table

## Implementation Details

### User Story 1 — Start an Exam Session (`POST /v1/exams`)

**Lambda**: `rubik-dev-exam-create` → `dist/exam-create.handler.js`  
**Handler**: `src/handlers/exam-create.handler.ts`  
**Service**: `src/services/exam.service.ts` → `createSession(examType)`

**Flow**:
1. Parse JSON body from `event.body`
2. Validate `examType` — must be present and equal `"TESTING"`. Return `400` on failure.
3. Generate `SK`: `${new Date().toISOString()}#${randomUUID().replace(/-/g,'').slice(0,4)}`
4. Call `exam.service.ts → createSession()`: `PutItem` to DynamoDB with `PK="EXAM_STATS"`, `SK`, `user="default@user.me"`, `examType`, `initialDatetime=SK.split('#')[0]`
5. Return `201 { sessionId: SK }`

**Error responses**:
- Missing/invalid `examType` → `400 { error: "examType must be one of: TESTING" }`
- DynamoDB error → `500 { error: "internal server error" }` (log full error)

---

### User Story 2 — Finish an Exam Session (`PUT /v1/exams/:sessionId`)

**Lambda**: `rubik-dev-exam-finish` → `dist/exam-finish.handler.js`  
**Handler**: `src/handlers/exam-finish.handler.ts`  
**Service**: `src/services/exam.service.ts` → `finishSession(sessionId, result)`

**Flow**:
1. Extract `sessionId` from `event.pathParameters.sessionId` (API Gateway decodes `%23` → `#` automatically)
2. Parse JSON body. Validate `result` — must be `"pass"` or `"fail"`. Return `400` on failure.
3. Call `exam.service.ts → finishSession()`:
   - `GetItem` with `PK="EXAM_STATS"`, `SK=sessionId`
   - If item not found → return `404 { error: "session not found" }`
   - Compute `finishDatetime = new Date().toISOString()`
   - Compute `timeSeconds = Math.floor((new Date(finishDatetime).getTime() - new Date(item.initialDatetime).getTime()) / 1000)`
   - `UpdateItem` setting `finishDatetime`, `timeSeconds`, `result`
4. Return `200 { status: "updated" }`

**Error responses**:
- Missing/invalid `result` → `400 { error: "result must be one of: pass, fail" }`
- Session not found → `404 { error: "session not found" }`
- DynamoDB error → `500 { error: "internal server error" }` (log full error)

### Shared Model (`src/models/exam-session.model.ts`)

```ts
export type ExamType = 'TESTING';
export type ExamResult = 'pass' | 'fail';

export const VALID_EXAM_TYPES: ExamType[] = ['TESTING'];
export const VALID_RESULTS: ExamResult[] = ['pass', 'fail'];

export interface ExamSession {
  PK: string;
  SK: string;
  user: string;
  examType: ExamType;
  initialDatetime: string;
  finishDatetime?: string;
  timeSeconds?: number;
  result?: ExamResult;
}
```

### DynamoDB Service (`src/services/exam.service.ts`)

Uses `DynamoDBDocumentClient` from `@aws-sdk/lib-dynamodb`.

```ts
// Table name from environment variable
```

- `createSession(examType, sessionId, initialDatetime)`: `PutCommand`

### Build Configuration

**`esbuild.config.js`**:
```js
const { build } = require('esbuild');

const handlers = ['exam-create', 'exam-finish'];

Promise.all(
  handlers.map(name =>
    build({
      entryPoints: [`src/handlers/${name}.handler.ts`],
      bundle: true,
      platform: 'node',
      target: 'node20',
      outfile: `dist/${name}.handler.js`,
      sourcemap: false,
      minify: false,
    })
  )
).catch(() => process.exit(1));
```
